# 硬件选型与接入建议：带屏 + 语音查天气 + 接入本物联网平台

本文面向要在**本仓库天气/物联网平台**上做硬件接入的同学，给出一套可落地的**采购与架构思路**（非商业广告，型号可自行检索购买）。

---

## 一、先理清两件事

### 1. 天气预报数据从哪来？

本平台的**区域天气预报**由后端调用 Open-Meteo，对外提供 HTTP 接口，例如：

- `GET /api/weather/search?q=城市关键词` — 搜地点，返回经纬度等  
- `GET /api/weather/current?lat=…&lon=…` — 取实况与约 7 日概况（与网页首页一致）

硬件只要 **能连 WiFi、能发 HTTP/HTTPS 请求**，即可访问你部署服务的主机（局域网 IP 或公网域名），**不必**在设备上跑完整浏览器。

### 2. 「语音查哪个地区」需要哪些环节？

典型链路是：

**麦克风 → 语音识别成文字 → 把文字当作城市名调用 `/api/weather/search` → 取第一条结果的经纬度 → 调用 `/api/weather/current` → 在显示屏上展示。**

在资源很紧的单片机上做**高准确率中文离线语音识别**通常不现实，因此常见做法是：**语音在云端或较强本地算力上识别**，小设备负责采集、上传或接收识别结果与展示。

---

## 二、和本平台的「接入」可以有两种

| 目标 | 做法简述 |
|------|----------|
| **只看预报 + 语音查城市** | 设备作为 **HTTP 客户端**，只调你服务器上的 `/api/weather/*`。 |
| **还要把传感器数据进平台** | 额外使用 **`POST /api/ingest`**，JSON 中带网页里注册设备得到的 **`device_token`**，以及 `temperature`、`humidity`、`pressure` 等字段（与项目文档、示例一致）。 |

可选：若服务端开启了 MQTT，也可按项目约定主题上报；课设阶段用 HTTP 往往更简单。

## 三、方案 A：少踩坑、快速出原型（适合新手）

### 思路

用 **树莓派（Linux 小电脑）** 同时驱动屏幕、麦克风，用 **Python** 调你自己的 Flask 接口；语音识别走**成熟云服务**或 API。

### 推荐采购（可按需增减）

| 物品 | 作用 |
|------|------|
| **树莓派 4B 或 5**（内存建议 2GB 起） | 运行系统与程序，网络稳定、资料多 |
| **7 寸 HDMI 触屏**（或官方 7 寸屏套件） | 展示天气、触摸选城市 |
| **USB 麦克风** 或 **带麦的 USB 摄像头** | 语音输入 |
| **官方或 5V/3A 以上合格电源** | 避免掉电、不稳定 |
| **MicroSD 卡**（32GB 级即可） | 安装 Raspberry Pi OS |

### 语音如何实现（软件层面）

在树莓派上用 Python：

- 调用 **百度 / 讯飞 / 阿里云** 等「一句话识别」类接口（常有试用额度）；或  
- 使用 **Whisper 类 API**（需网络与对应账号策略）。

识别得到城市名字符串后，拼 URL 请求：

`http://<你的服务器>:5000/api/weather/search?q=…`

再解析 JSON，用返回的 `latitude`、`longitude` 请求：

`http://<你的服务器>:5000/api/weather/current?lat=…&lon=…`

与浏览器首页逻辑一致。

### 优缺点

- **优点**：开发快、语音方案成熟、调试方便（ssh、打印日志）。  
- **缺点**：成本高于单片机、体积与功耗更大。

## 四、方案 B：偏智能硬件课设、成本较低

### 思路

**ESP32-S3** + **SPI/I8080 彩屏** + **I2S 数字麦克风**。设备负责联网、显示；语音可走「录音上传到你电脑/树莓派上的转写服务」或云端 STT，再回传文字到设备查天气。

### 推荐采购（示例组合）

| 物品 | 说明 |
|------|------|
| **ESP32-S3 开发板**（带 WiFi，Flash/PSRAM 按商家说明选） | 比经典 ESP32 更适合带屏与较复杂逻辑 |
| **2.4～3.5 寸彩屏**（常见驱动如 ILI9341、ST7789 等，注意 3.3V 与接线） | 显示城市、实况温度、湿度等文字或小图标 |
| **I2S 麦克风模块**（如 INMP441、SPH0645 等） | 数字麦克风，接 ESP32 的 I2S 引脚 |

### 语音注意点

- 在 ESP32 本机跑完整中文离线识别**不推荐作为课设第一步**。  
- 可行路径：**短录音 → WiFi 发到树莓派/PC 上的转写服务 → 返回文本 → ESP32 再调天气接口**；或使用厂商提供的**语音云 + SDK**（需评估中文文档与账号）。

### 优缺点

- **优点**：成本低、像典型「物联网终端」、可同时 **POST `/api/ingest`** 上报传感器。  
- **缺点**：语音链路需自己拼，整体调试时间通常长于树莓派方案。

## 五、方案 C：折中（一体化开发板）

部分 **ESP32-S3 带屏、甚至带麦/喇叭的一体板**（各厂商「开发盒 / Box」类）可减少飞线，适合做「带屏的小终端」演示。

购买前建议确认：

- 是否提供 **WiFi + HTTP 客户端** 示例；  
- 语音是否为 **中文** 场景、是否有可用示例或必须绑定的云服务。

---

## 六、网络与部署提醒

- **仅在同一 WiFi 下演示**：硬件里把服务器地址写成你电脑的 **局域网 IP**，例如 `http://192.168.1.x:5000`。  
- **硬件在外网、服务器在家**：需要 **内网穿透 / 公网 IP / 云服务器**，并注意 **HTTPS** 与防火墙端口。  
- 语音若走**公网云厂商**，设备需能访问外网；仅局域网时，可把「转写」放在与你 Flask **同一局域网**的一台小电脑上。

---

## 七、采购清单小结（按目标选）

| 你的侧重点 | 建议 |
|------------|------|
| 优先做出来、语音识别要稳 | **树莓派 + HDMI 屏 + USB 麦** |
| 预算紧、突出传感器 + 屏显 | **ESP32-S3 + 小彩屏**，语音第二阶段再接云端或中转服务 |
| 想要一体化、少飞线 | **ESP32-S3 类一体开发盒**，务必核对中文语音与文档 |
