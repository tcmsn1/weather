# 云感天气平台 — 分享给同学时的部署说明

把本文件夹（或 Git 仓库）交给同学后，按下面步骤可在**另一台电脑**上正常运行。默认以 **Windows** 为例；若使用 Mac / Linux，把文中 PowerShell 命令换成对应终端命令即可。

---

## 一、同学电脑上需要事先安装的东西

| 软件 | 用途 | 说明 |
|------|------|------|
| **Python 3.10+** | 运行后端 | 安装时勾选 **Add Python to PATH**。若只有 `py` 命令，下文把 `python` 改成 `py` 即可。 |
| **MySQL 8.x** | 存用户、设备、上报数据 | 安装 **MySQL Server**，记住自己设的 **root 密码**（或单独建库 + 专用账号，见下文）。 |
| **能上网** | 区域天气预报 | 首页预报走 **Open-Meteo** 在线接口，无外网则预报不可用，但登录与设备功能仍可用。 |

可选：**MQTT 客户端 / Mosquitto**（只有要用 MQTT 上报时才需要）。

---

## 二、把项目交给同学的几种方式

任选其一即可：

1. **压缩包**：把整个项目文件夹打成 `.zip` 发给对方（**不要**打包 `.venv`，体积大且机器不同容易坏）。  
2. **Git**：把项目推到 GitHub / Gitee 等，把仓库地址发给对方，`git clone` 即可。

**不要**把你自己电脑上的 **数据库密码、SECRET_KEY** 写进代码或公开仓库里。每人应在自己电脑上单独配置环境变量。

## 三、同学拿到代码后的操作步骤

### 1. 解压 / 克隆到本地

例如目录为：

`D:\projects\iot-weather-platform`

下文均假设已 `cd` 到**项目根目录**（能看到 `app.py`、`requirements.txt`）。

### 2. 创建虚拟环境（推荐）

PowerShell：

```powershell
cd D:\projects\iot-weather-platform
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

若提示无法执行脚本，可先执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### 3. 安装 Python 依赖

```powershell
python -m pip install -r requirements.txt
```

若下载慢，可使用清华镜像（任选）：

```powershell
python -m pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 4. 在 MySQL 里准备数据库

用 **MySQL Workbench**、**命令行** 或任意客户端执行（把密码改成同学自己的）：

```sql
CREATE DATABASE IF NOT EXISTS weather_iot
  DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 方式 A：继续用 root（仅适合课设）
-- 只需保证 weather_iot 库存在即可。

-- 方式 B：专用账号（更安全，推荐写进报告）
CREATE USER IF NOT EXISTS 'iot'@'%' IDENTIFIED BY '同学自己设一个密码';
GRANT ALL PRIVILEGES ON weather_iot.* TO 'iot'@'%';
FLUSH PRIVILEGES;
```

确认 **MySQL 服务已启动**（Windows「服务」里可看到 MySQL）。

### 5. 配置环境变量并启动

在**同一 PowerShell 窗口**里（已激活 `.venv`）执行，**把密码等改成同学自己的**：

```powershell
$env:MYSQL_USER="root"
$env:MYSQL_PASSWORD="同学的MySQL密码"
$env:MYSQL_DATABASE="weather_iot"
$env:SECRET_KEY="一串足够长的随机英文字符用于会话加密"
python app.py
```

若使用专用账号 `iot`，则：

```powershell
$env:MYSQL_USER="iot"
$env:MYSQL_PASSWORD="iot账号的密码"
$env:MYSQL_DATABASE="weather_iot"
$env:SECRET_KEY="一串随机字符"
python app.py
```

看到类似 `Running on http://127.0.0.1:5000` 即成功。

#### 管理员（可选）

若需要能**查看全站所有用户设备**的账号：

1. 先在网页上 **注册** 一个用户名（例如 `admin`）。  
2. 在启动前增加环境变量（多个用英文逗号分隔，**不区分大小写**）：

```powershell
$env:ADMIN_USERNAMES="admin,teacher"
python app.py
```

3. 控制台会提示已将匹配用户设为管理员；该用户需 **退出并重新登录** 后，导航栏会出现「管理员」标签，首页「我的气象站」会变为 **全站设备** 列表。

取消管理员可在 MySQL 中执行：`UPDATE users SET is_admin=0 WHERE username='xxx';`  
**注意**：`ADMIN_USERNAMES` 每次启动都会把名单中的用户**提升**为管理员；不要在不受信任的环境把管理员名写进公开配置。

### 6. 浏览器访问

- 首页：<http://127.0.0.1:5000/>  
- 数据库是否正常：<http://127.0.0.1:5000/api/health>  

先 **注册账号**，再添加设备；硬件上报仍用接口文档里的 `device_token`。

## 四、让同一局域网其他同学访问（可选）

你的电脑运行 `python app.py` 后，控制台会显示本机局域网 IP，例如：

`http://192.168.x.x:5000`

对方在同一 WiFi 下可用该地址访问。**Windows 防火墙**可能拦截 5000 端口，需在「防火墙 → 入站规则」里允许 `python.exe` 或端口 **5000**，否则别人打不开。

## 五、常见问题

### 1. 提示 `Python was not found`

安装 Python 并勾选 **Add to PATH**，或改用：

```powershell
py -m venv .venv
py app.py
```

### 2. `ModuleNotFoundError: No module named 'pymysql'`

先激活 `.venv`，再执行：

```powershell
python -m pip install -r requirements.txt
```

### 3. `Access denied ... using password: NO`（错误 1045）

说明**没带上 MySQL 密码**。在同一窗口设置 `MYSQL_PASSWORD` 后再启动（见上文第五节）。

### 4. 预报一直加载失败

检查**能否访问外网**；部分校园网会拦截国外 API。设备上报与登录不依赖预报接口。

### 5. 关掉 PowerShell 后又要重新设环境变量

是的，**环境变量只对当前窗口有效**。可把多行 `$env:...` 存成 `env.ps1`，每次进项目先执行：

```powershell
.\env.ps1
python app.py
```

**不要把含真实密码的 `env.ps1` 上传到 Git**；可只发 `env.example.ps1` 模板给同学，让对方复制改名后自行填写。

## 六、交给同学时的「最小检查清单」

- [ ] 已安装 Python 3.10+ 且能执行 `python --version`  
- [ ] 已安装 MySQL 且 `weather_iot` 库已创建  
- [ ] 已在项目目录创建 `.venv` 并 `pip install -r requirements.txt`  
- [ ] 已设置 `MYSQL_USER`、`MYSQL_PASSWORD`、`MYSQL_DATABASE`、`SECRET_KEY`  
- [ ] （可选）需要管理员时设置 `ADMIN_USERNAMES` 并重新登录  
- [ ] `python app.py` 无报错，浏览器能打开首页并完成注册  

按上述做完，同学即可在他自己的电脑上独立运行同一套系统。

## 七、数据与版权说明（可写进报告）

- **区域预报数据**来自 [Open-Meteo](https://open-meteo.com/)（免费接口，有合理更新延迟）。  
- **物联网数据**由各自设备上报，仅存于同学自己安装的 MySQL 中，互不共享。

如有课程要求「部署截图」，可截取：终端启动成功、首页预报、注册登录、设备卡片、`/api/health` 返回 `ok: true` 等页面。
